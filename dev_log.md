## 記録日時: 2026-03-10 17:55

### 成功の境界線

- **起動速度の極限短縮 (3秒から約1秒へ)**:
  - `main.py` のトップレべルで行っていた `setup_logging` (ファイルオープン) を遅延させ、エラー時または設定画面起動時のみ実行するように変更。
  - `os.path.exists` によるファイルチェックを UI スレッド初期化後に移動し、プロセスの垂直立ち上げを優先。
  - 関連付け起動時の「重さ」が完全に解消された。
- **確実なフォーカス奪取とキーボード操作**:
  - `launcher_ui.py` の `_force_focus` を強化。Windows API (`SetForegroundWindow`) を 100ms 間隔で最大5回リトライする仕組みを導入。
  - これにより、Explorer (親プロセス) がフォーカスを保持しようとする挙動を上書きし、起動直後の Tab キー操作を可能にした。
- **ビルドの恒常化**:
  - 改変後に必ず `pyinstaller` によるビルドを実行し、実行ファイル (`.exe`) 形式での動作を毎回保証する運用を開始。

## 結論

「3秒の遅延」という具体的課題に対し、ディスクI/Oの徹底的な遅延と、OSのフォーカス制御に対する積極的なアプローチ (Retry API) で解決した。Tabキーが効かない原因が「ウィンドウがまだフォーカスを得ていない」ことにあったため、これを修正したことで操作性が劇的に改善。
