# AppSelecter Web版コンセプト

先ほど試作したWeb版が「どういう動き」をするものなのか、デスクトップ版との違いを説明します。

---

## Web版でできること / できないこと

| 機能 | デスクトップ版 (Python) | Web版 (ブラウザ) |
|------|:---:|:---:|
| ボタン一覧の表示 | ✅ | ✅ |
| ボタン名・パスの編集 | ✅ | ✅ |
| 拡張子に紐づいた自動起動 | ✅ | ❌ |
| ローカルの .exe を起動 | ✅ | ❌ |
| URLをブラウザで開く | ✅ | ✅ |
| 設定の永続保存 | ✅ (JSON) | ✅ (localStorage) |
| オートクローズタイマー | ✅ | △ (シミュレーション) |

---

## なぜWeb版では「ローカルアプリの起動」ができないのか

ブラウザはセキュリティ上の理由から、**Webページからユーザーのパソコン内のプログラムを直接起動することを禁止**しています。

もしそれが許可されていたら、悪意のあるWebサイトがユーザーのパソコン内のプログラムを勝手に実行できてしまいます。そのため、ブラウザは「Webの世界」と「ローカルのパソコンの世界」を厳格に分けています。

---

## Web版の活用例（もし発展させるなら）

Web版でも意味があるのは、**「URLランチャー」としての使い方**です。

- よく使うWebサービス（Google Drive, Notion, GitHub等）へのショートカット集
- チーム共有のリンク集（URLをJSONやDBに保存して共有）
- PWA（Progressive Web App）としてスマートフォンのホーム画面に追加

---

## まとめ

今回のAppSelecterの目的（**拡張子に紐づけて、ローカルのアプリを選んで起動する**）を実現するには、**デスクトップ版（Python版）一択**です。

Web版は「お気に入りリンク集」のような使い方であれば有効ですが、今回のユースケースには当てはまりません。
先ほど作成したWeb版は、UIの実験（デザインの方向性確認）として活用いただけます。
